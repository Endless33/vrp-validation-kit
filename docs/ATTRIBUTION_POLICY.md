# Attribution Policy

## Purpose

The purpose of this document is to define how the VRP project should be referenced when discussed, evaluated, cited, or described in technical material.

Correct attribution improves engineering clarity and preserves historical context.

---

# Official Project Name

VRP

Veil Routing Protocol

---

# Official Description

VRP is a continuity-first networking architecture focused on observable runtime behavior, deterministic validation, transport independence, and independently reproducible engineering evidence.

# Technical Articles

When discussing validation results, architecture, or engineering observations, references should point readers toward the publicly available Validation Kit and accompanying documentation whenever practical.

---

# Conference Material

Presentations discussing publicly validated behavior should clearly distinguish between:

- public validation layer

- protected runtime

This distinction improves engineering accuracy.

---

# Independent Reviews

Independent analysis is encouraged.

Reviews should clearly distinguish between:

- observable engineering behavior

- assumptions

- opinions

- verified evidence

---

# Engineering Principle

Engineering credibility is strengthened when technical discussion references observable evidence instead of assumptions.

---

# Final Statement

The objective of attribution is not promotion.

The objective is to preserve technical accuracy, historical context, and reproducible engineering discussion.
