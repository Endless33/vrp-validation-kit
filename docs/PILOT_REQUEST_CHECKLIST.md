# VRP Pilot Request Checklist

Before requesting a Pilot, prepare the following:

- organization name;
- technical contact;
- target environment;
- expected use case;
- network conditions to validate;
- failure scenarios of interest;
- preferred evaluation timeline;
- expected success criteria.

## Minimum Technical Scope

The Pilot request should identify at least one of the following:

- transport migration;
- blackout recovery;
- session continuity;
- replay rejection;
- evidence verification;
- path instability;
- multi-node coordination;
- degraded network operation.

## Request Quality

Requests without a defined environment or validation objective may be rejected or deferred.

The Pilot is intended for serious technical evaluation, not informal product demonstrations.
