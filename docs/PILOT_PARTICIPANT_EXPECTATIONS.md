# VRP Pilot Participant Expectations

## Suitable Participants

The Pilot is intended for organizations evaluating:

- resilient transport systems;
- distributed runtimes;
- continuity-first networking;
- edge infrastructure;
- cloud and hybrid network environments;
- robotics or autonomous systems;
- critical communication paths.

## Participant Responsibilities

The participant should be able to provide:

- a controlled test environment;
- at least one technical contact;
- agreed failure scenarios;
- execution time for validation;
- feedback based on observable behavior.

## What the Participant Can Do

Participants may:

- interrupt transport paths;
- inject delay and packet loss;
- trigger path migration;
- test recovery behavior;
- verify exported evidence;
- compare declared and observed verdicts.

## What the Participant Must Not Expect

The Pilot does not include:

- ownership of the private runtime;
- transfer of proprietary algorithms;
- unrestricted internal instrumentation;
- access to protected source code;
- permission to reproduce the private architecture independently.

## Engagement Principle

The Pilot evaluates behavior.

It does not transfer the architecture.
